Walkthrough 1Simple regression: does distance predict trade?

One predictor. You do not even need a regression command. Three built-in functions do it. Put these on a blank part of any sheet (we will use cells next to each label).

1
The slope (the distance elasticity). Type:
=SLOPE(ln_trade, ln_dist)
You should see …. Trade falls as distance rises, which is the negative sign. Because both sides are logged, this is an elasticity: a 1% rise in distance goes with about a … change in trade.
2
The intercept and R-squared:
=INTERCEPT(ln_trade, ln_dist) =RSQ(ln_trade, ln_dist)
Intercept …, R-squared …. That R-squared is the headline: distance alone explains only about … of why trade volumes differ. A real effect that still leaves almost everything unexplained. That is the cue for Walkthrough 2.
3
Is the slope real? Get its t-statistic. The slope's standard error hides in the LINEST block:
=INDEX(LINEST(ln_trade, ln_dist, TRUE, TRUE), 2, 1)
Standard error …. Divide the slope by it for the t-stat: …. As a rule of thumb, |t| above 2 means the effect is unlikely to be an accident.
4
The named ranges. Those names (ln_trade, ln_dist) already exist in the workbook, which is why the formulas read so cleanly. You can also point at the raw columns: Data!I2:I201 for ln_trade, Data!J2:J201 for ln_dist.

Walkthrough 2Multiple regression: the gravity equation

Add the two economies' sizes. With more than one predictor, SLOPE will not serve. The tool is LINEST, which returns a whole block at once.

1
Select a 5-row by 4-column block first (for 3 predictors you get 4 columns), then type the formula and press Ctrl+Shift+Enter on older Excel, or just Enter on Excel 365:
=LINEST(ln_trade, X_grav, TRUE, TRUE)
X_grav is the named block Data!J2:L201 = ln_dist, ln_gdp_i, ln_gdp_j sitting in three adjacent columns. LINEST needs the X columns to be next to each other, which is why the Data tab is arranged that way.
2
Mind the reversal. LINEST returns coefficients right to left, with the intercept last. So for X_grav the top row reads: ln_gdp_j, then ln_gdp_i, then ln_dist, then the intercept. Pull each one cleanly with INDEX:
=INDEX(LINEST(ln_trade,X_grav,TRUE,TRUE),1,3) 'ln_dist coef =INDEX(LINEST(ln_trade,X_grav,TRUE,TRUE),1,2) 'ln_gdp_i coef =INDEX(LINEST(ln_trade,X_grav,TRUE,TRUE),1,1) 'ln_gdp_j coef =INDEX(LINEST(ln_trade,X_grav,TRUE,TRUE),1,4) 'intercept
You should get ln_dist …, ln_gdp_i …, ln_gdp_j …. Both GDP elasticities land near 1, exactly what gravity predicts.
3
R-squared, the payoff. It sits in row 3, column 1 of the block:
=INDEX(LINEST(ln_trade,X_grav,TRUE,TRUE),3,1)
R-squared jumps to …, up from about 0.06 with distance alone. Economic size was the missing piece. Each coefficient now reads as a partial effect: what that variable does holding the others fixed.

Walkthrough 3Dummy variables: borders, language, trade deals

Not everything is a quantity. A dummy is 1 when something is true and 0 when it is not. You add it to the regression like any other column. Because trade is in logs, a dummy's coefficient reads as a percentage gap between the two groups.

1
Run the full model over six predictors. The block is now 5 rows by 7 columns:
=LINEST(ln_trade, X_full, TRUE, TRUE)
X_full = Data!J2:O201, which adds contiguity, common_language and agree_fta to the three logged variables.
2
Read a dummy as a percentage. The trade-agreement coefficient is the last X column, so it is column 1 of the LINEST block. Pull it, then convert:
=INDEX(LINEST(ln_trade,X_full,TRUE,TRUE),1,1) 'agree_fta coef =EXP(INDEX(LINEST(ln_trade,X_full,TRUE,TRUE),1,1))-1 'exact % effect
The coefficient is about …. A small coefficient is roughly coef × 100 percent; for larger ones use the exact EXP(b)-1, which here is about …. A trade agreement goes with trade more than tripling.
3
When a sign surprises you. The shared-border coefficient comes out …, which looks backwards. With only 200 pairs the few neighbors are also short-distance and high-GDP, so once those are controlled the leftover border term is estimated off very little. The lesson is to get more data and look harder, not to report that borders cut trade. Real data argues back.
